## 1. アプリの目的・設計思想

### 何のために作るか
山本輝（ヒカヤマ）個人が、仕事・プライベートの情報を一元管理するパーソナルダッシュボード。
従来はClaudeのチャット画面でタスクや予定を管理していたが、チャットの投稿制限・引き継ぎの手間・マルチデバイス対応の不便さを解消するために作る。

### 核心となる設計思想
**「雑に投げてAIが整形する」入力体験**
- テキストで雑に投げる（例：「来週火曜3時にくみこさんとMTG」）→ ClaudeのAPIが構造データに変換して登録
- 画像・スクショを投げる → Claude Vision APIで内容を読み取り予定を抽出・登録
- 確認ステップを1回挟み、ワンクリックで登録（誤登録防止）

**「データはアプリが持つ、文脈はClaude.mdが持つ」**
- Claudeのチャット履歴には依存しない
- 「Claudeに聞く」ボタンを押したとき、全データを自動でプロンプトに埋め込んでAPIに投げる
- ユーザーが毎回コンテキストを説明しなくて済む設計

### 将来的な拡張方針
- フェーズ2：Googleカレンダー連携（選択式で予定を送信）
- フェーズ2：Firebase Hostingへ移行（現在はGitHub Pages）
- フェーズ3：チームへの展開（現在は個人利用のみ）

## 2. 技術スタック

| 項目 | 採用技術 | 理由 |
|------|----------|------|
| フロントエンド | React（Vite） | proline-dashboardと同じ構成、慣れている |
| データベース | Firebase Firestore（新プロジェクト） | リアルタイム同期・マルチデバイス対応 |
| ホスティング | GitHub Pages | 無料・シンプル・個人利用に最適 |
| AI連携 | Anthropic API（claude-sonnet-4-20250514） | テキスト解析・Vision・AI問い合わせ |
| 認証 | Firebase Auth（Googleログイン） | 個人利用のためシンプルに |
| CI/CD | GitHub Actions | GitHub Pagesへの自動デプロイ |

### Firebaseプロジェクト
- 新規プロジェクトを作成する（proline-dashboardとは分離）
- プロジェクト名候補：`hikayama-dashboard`
- Firestoreのリージョン：asia-northeast1（東京）

## 3. データ構造（Firestoreスキーマ）

### コレクション設計

```
/users/{userId}/
  /tasks/{taskId}
    - title: string          // タスク名
    - status: string         // "todo" | "in_progress" | "done"
    - deadline: timestamp    // 締切日時
    - assignee: string       // 担当者名（省略可）
    - notes: string          // メモ（省略可）
    - tags: string[]         // タグ（省略可）
    - createdAt: timestamp
    - updatedAt: timestamp

  /events/{eventId}
    - title: string          // 予定名
    - type: string           // "work" | "private" | "shift"
    - startAt: timestamp     // 開始日時
    - endAt: timestamp       // 終了日時
    - location: string       // 場所（省略可）
    - syncToGcal: boolean    // Googleカレンダーに同期するか（フェーズ2用）
    - notes: string
    - createdAt: timestamp
    - updatedAt: timestamp

  /memos/{memoId}
    - content: string        // メモ本文
    - category: string       // "work" | "private" | "idea"
    - createdAt: timestamp
    - updatedAt: timestamp
```

## 4. 画面構成

### メイン画面（3ペイン構成）
```
┌─────────────────────────────────────────────┐
│  ヘッダー：アプリ名 / 雑に入力ボタン / Claudeに聞くボタン  │
├──────────┬──────────────────┬───────────────┤
│ タスク一覧  │  カレンダービュー   │  メモ・状況報告  │
│（ステータス・│ （今日/今週）      │               │
│  締切フィルタ）│ 仕事シフト＋      │               │
│          │ プライベート統合   │               │
└──────────┴──────────────────┴───────────────┘
│  「Claudeに聞く」チャットパネル（下部・開閉式）          │
└─────────────────────────────────────────────┘
```

### 雑入力モーダル
- テキスト入力エリア（複数行）
- 画像・スクショアップロードエリア（ドラッグ&ドロップ対応）
- 「解析する」ボタン → Claude APIに投げる
- 解析結果の確認画面 → 「登録する」ワンクリックで保存

### 「Claudeに聞く」パネル
- 質問入力欄
- 送信時に全タスク・予定・メモをシステムプロンプトに自動挿入
- 例：「今週の優先順位は？」「明日何がある？」「遅れているタスクは？」

## 6. AI連携の実装方針

### 雑入力の解析プロンプト（テンプレート）
```
あなたはヒカヤマさんのパーソナルアシスタントです。
以下の入力から、タスク・予定・メモを抽出してJSON形式で返してください。

入力：{ユーザーの雑なテキストまたは画像の内容}

出力形式：
{
  "tasks": [...],
  "events": [...],
  "memos": [...]
}

不明な項目はnullにしてください。推測で補完しないでください。
```

### 「Claudeに聞く」プロンプト（テンプレート）
```
あなたはヒカヤマさん（merone社アクセラチームリーダー）のパーソナルアシスタントです。
以下のデータを参照して質問に答えてください。

【現在のタスク一覧】
{tasks_json}

【今後の予定】
{events_json}

【最近のメモ】
{memos_json}

質問：{ユーザーの質問}
```
